Auto, or open the app?

A Mumbai ride-fare decision tool that's honest about when it actually knows the answer.

Mumbai's auto-rickshaw meter is government-fixed and published — exact, surge-proof, no guessing. Uber/Ola/Rapido pricing is proprietary and dynamic — it can only ever be estimated. Most tools blur that line and hand you a confident number either way. This one doesn't: a strong verdict when the gap is wide, a plain "toss-up" when it isn't — with the surge-proof meter as your anchor exactly when app prices are least trustworthy.

How it works

Three steps — the two that touch money are deterministic.

1

Exact meter math

₹26 minimum (first 1.5km) + ₹17.14/km, +25% night — the actual Maharashtra RTO tariff, computed exactly on the real road distance. No model, no guess.

2

Modeled app fares + predicted surge

App fares estimated from published rate cards. Surge isn't guessed by you — it's predicted from time, day, and live weather, and clearly labeled as a prediction.

3

Confidence-calibrated verdict

The gap between meter and cheapest app option decides confidence — not the rupee amount. Small gap = "toss-up," not a false-sure recommendation.

Tech stack

Each layer does one job — the LLM is never the one doing arithmetic.

Routing

OpenStreetMap (free)

Photon geocoding + OSRM road distance — any Mumbai route, no Google, no billing account, no key. No-auto zone is a point-in-polygon test.

Fare + Surge

Deterministic Python

Official RTO meter formula + modeled app rate cards. Surge predicted from time + day + live weather (keyless Open-Meteo), labeled as a prediction.

Decision Logic

Confidence engine

Gap-based confidence, surge-proof anchoring, zone handling, and a car-tier "don't default to the cab" call-out. Never sounds surer than it is.

NL Assistant + API

Claude (Haiku) · FastAPI

Claude parses questions and explains verdicts — never computes a fare. FastAPI serves /decide_route, /ask, deployed on Render.

Evals

How the tool's confidence calibration was tested.

A

Decision engine: 22/22

Verdict + confidence on 22 route/time/surge combinations checked against an independently re-derived RTO formula — not just tested against its own code.

B

High-confidence precision: 100%

Primary metric. Of every route the NL assistant matched with confidence, 15/15 were correct (live Claude calls). A wrong confident match is the failure mode this project exists to prevent.

C

Correct refusal: 6/6

Out-of-scope queries (wrong city, gibberish, an unsupported South Mumbai pair) were all correctly declined instead of guessing a plausible wrong route.

Honest caveat: 21 NL test cases validates the design, not bulletproof reliability. Full per-case results in evals/results.md.
